# sema-core — Veric Output Contract

sema-core defines the rkyv types for the verified program
produced by veric and consumed by semac, domainc, rsc, askid.

The naming follows the pattern: semac produces sema, and
sema-core is the core contract for the sema stage — the
verified input that semac works from.


## Types

From `core/program.aski` (generated by corec):

- **TypeKind** — Enum, Struct, Newtype, Const
- **TypeLocation** — module index + kind + definition index
- **TraitLocation** — module index + declaration index
- **ResolvedImport** — name + resolved TypeLocation
- **ImportResolution** — per-module resolved imports

These are the resolution table types. The actual ModuleDef
(with enums, structs, etc.) comes from the aski crate.
veric depends on both aski and sema-core. Its output
program.rkyv contains Vec<aski::ModuleDef> + resolution
table using sema-core types.


## The Three Contracts

```
aski-core (corec-generated) — askicc↔askic grammar types
aski     (corec-generated) — askic↔veric parse tree types
sema-core (corec-generated) — veric↔semac resolution types
```


## Regenerate

```
corec core generated/sema_core.rs
```
